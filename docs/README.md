# Documentation Guide

This directory contains supporting documentation for maintainers, contributors, and evaluators.

## Core references

- `ARCHITECTURE.md`: runtime flow and module boundaries
- `RATIONALE.md`: key design decisions and tradeoffs
- `CITATIONS.md`: external references used by the project
- `OPERATIONS_RUNBOOK.md`: practical runbook for local operation and incident response

## Release and community docs

- `SEED_ISSUES.md`: contribution-friendly issue ideas
- `ECOSYSTEM_IMPACT_EVIDENCE.md`: adoption and ecosystem evidence tracking
- `LAUNCH_POSTS.md`: launch copy drafts for distribution channels
- `PUBLISH_CHECKLIST.md`: checklist for release and publication

## Historical planning artifacts

Planning artifacts used during early project setup were intentionally removed from the public docs surface to keep this repository focused on long-term maintainability and operator value.
